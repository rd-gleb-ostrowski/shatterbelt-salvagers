I am organising a Agentic Code Retreat, where beginners and intermediate developers
have one day to deepen their skill & understanding of working with LLM Agents.
Some participants will bring their own projects and tasks to work on, some will
even mentor others.
There will be a sizeable portion who never worked with an agent before
or have only basic knowledge, though.
For those specifically, I want to provide an example project to work on and
sharpen their skills.
The general agenda should also support working on this project and
give opportunity for short 'what went good, what went bad' talks after each session,
similar to how a regular code retreat is structured.
As beginners have no idea, it would also be important to not overwhelm them with
infomation; maybe start just by telling them to have a go with the prompt,
before introducing the plan mode 1 hour later, and then skills & mcp 1 hour after that
and finally a brief intro into OpenSpec, Superpower, GDS, drill-me etc.
like skills (the time in between session is just an example).
It would also be best, if in the morning the project/work can happen greenfield,
but shift to brownfield in the afternoon (for example, by swapping the generated
code in the morning between teams).
I also had the idea of the project begin some kinda competition, like for
example the AI bot MMOs (like Robocode Tank Royale),
where there's a central server running bot matches,
and participants program the AI for those bot and let them fight against each other.
But really, I have no idea how to best approach this, agenda and project wise,
and need some brainstorming ideas.

As a prerequisite, every participants will already have a agent,
most likely Copilot CLI, set up on their machines, so the retreat can skip that.
Sandboxing agents, skill marketplaces, MCP server security etc. are also
out of scope for the retreat.
